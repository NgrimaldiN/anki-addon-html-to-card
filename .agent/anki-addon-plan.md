# Anki Add-on Execution Plan

## Objective

Build an Anki add-on that adds a `Paste LLM Cards` action to the Add Cards window, lets the user paste one JSON bundle generated by an LLM, optionally creates or updates a styled note type, and adds one or many notes directly into the currently selected deck.

## Decisions Locked In

- Use JSON bundles, not executable pasted code.
- Use the currently selected Add Cards deck as the only deck target.
- Support bulk note creation in one operation.
- Support optional note type creation or update when the bundle includes `note_type`.
- Keep Anki-specific UI code thin and push most logic into testable pure-Python modules.

## Planned File Layout

- `__init__.py`
- `addon/entry.py`
- `addon/dialog.py`
- `addon/parser.py`
- `addon/schema.py`
- `addon/importer.py`
- `addon/errors.py`
- `tests/test_parser.py`
- `tests/test_importer.py`
- `README.md`
- `manifest.json`

## Architecture

- `entry.py`: register `add_cards_did_init`, inject the button, open the paste dialog.
- `dialog.py`: show a text area, parse button, import button, and result/error surface.
- `parser.py`: extract JSON from raw text or fenced code blocks.
- `schema.py`: normalize and validate bundle payloads into typed structures.
- `importer.py`: apply a validated bundle to Anki collection abstractions.
- `errors.py`: user-facing validation and import exceptions with precise messages.

## TDD Sequence

1. Write parser tests for raw JSON and fenced JSON.
2. Write schema tests for missing keys, unknown keys, and field mismatches.
3. Write importer tests for:
   - using the current note type
   - creating a new note type
   - updating an existing note type
   - adding multiple notes into one selected deck
4. Implement the pure-Python modules to satisfy the tests.
5. Add the Anki UI integration last.

## Risks

- Note type updates can become destructive if they silently rewrite user templates. The first version should update only the supplied note type and only the specific fields/templates declared in the bundle.
- Anki API drift across versions is possible. Favor current snake_case APIs and defensive imports.
- Large imports should run through `CollectionOp` so the Add Cards window stays responsive.

## Definition of Done

- User sees a new button in Add Cards.
- User can paste an LLM JSON bundle.
- The add-on creates one or many notes in the currently selected deck.
- The add-on can create a styled note type when requested.
- Validation errors are understandable.
- Core logic is covered by automated tests.
